# Log de Sesión: 2026-07-05

## 📝 Resumen de la Sesión
Preparación pre-producción: smoke test enfocado en **resiliencia offline**. Se auditó el código
real (no solo la spec) del flujo del mercaderista y se corrió la suite de tests (18/18 verdes al
inicio). **Hallazgo clave:** los flujos de *acción* (empezar ruta, check-in, finalizar) ya son
correctamente no-bloqueantes offline (escriben a SQLite, sync fire-and-forget). El **único hueco**
que puede trabar al mercaderista es que **cargar la ruta requería conexión** (`routesApi` pega a
Supabase sin caché) → cold-start offline dejaba pantalla de error sin poder empezar. Se implementó
el **caché offline de ruta** para cerrarlo. Smoke test en dispositivo pendiente (dev client Android).

## 🛠️ Cambios Realizados
- **`mobile/src/services/routeCache.ts`** (nuevo): snapshot de solo-lectura (ruta + tiendas) en la
  tabla `meta` de SQLite. `resolveRouteLoad` (puro, network-first con fallback), `save/loadRouteSnapshot`,
  `parse/serializeSnapshot` (null-safe). 6 tests TDD.
- **`mobile/src/context/RouteContext.tsx`**: `loadRoute` pasa a **network-first con fallback a caché**.
  Online OK → hidrata + guarda snapshot (fire-and-forget); offline con caché → hidrata desde snapshot
  y marca `routeFromCache`; offline sin caché → error honesto. Nuevo estado `routeFromCache`.
- **`mobile/src/screens/RouteScreen.tsx`**: banner sutil *"Modo offline · ruta en caché del {fecha}"*
  cuando se sirve desde caché. No bloquea ninguna acción.
- **`mobile/src/sync/bannerState.ts`** (nuevo) + **`SyncBanner.tsx`**: banner de sync con estados
  claros por tono — *"Sincronizando N…"* (navy), *"Sin conexión · N en cola"* (ámbar), *"N registros
  por subir"* (ámbar), oculto cuando no hay nada pendiente. Función pura `deriveSyncBanner` derivada
  de `SyncContext` (`status` + `pendingCount`), **sin detección de red nativa** (JS puro, sin rebuild).
  8 tests TDD. Decisión: NO se agregó NetInfo/expo-network para evitar recompilar dev client + APK.
- **`mobile/src/utils/withTimeout.ts`** (nuevo) + `RouteContext.startSession`: "Empezar Ruta" arranca
  en ~3s (fix GPS precisión Balanced con timeout; antes ~15s por `getCurrentPositionAsync` de alta
  precisión sin timeout). 4 tests.
- **Ícono de app nuevo** — "Origen → Destino": aro de origen + trayecto punteado + pin de destino
  verde sobre navy de marca. `mobile/assets/icon.png` (navy completo) + `adaptive-icon.png` (foreground
  transparente), `app.json` `adaptiveIcon.backgroundColor` → `#00205C`. Rasterizado del SVG con `sharp`
  (herramienta desechable en scratchpad), verificado con preview de máscara circular (zona segura OK).
- **Diseño**: `docs/superpowers/specs/2026-07-05-cache-offline-ruta-design.md` (local, path ignorado por convención).

## 🤝 Decisiones Tomadas
- *Storage del caché*: snapshot JSON en la tabla `meta` existente (cero tablas nuevas, cero FKs) en
  vez de tablas espejo — mínimo riesgo pre-producción.
- *Estrategia*: network-first con fallback (online siempre gana; la fuente de verdad del plano de
  lectura sigue siendo Supabase, ver [[arquitectura/Spec - Rediseño Motor de Sync Móvil]] §3.1).
- *YAGNI*: sin expiración de caché (se muestra el último snapshot con su fecha visible) y sin caché
  de marcas de competencia (follow-up).

## 🐛 Smoke Test en APK (build `677959a`) — hallazgos

- **Empezar Ruta funciona online** (evidencia `device_logs`: sesión `063b6533` con 2 pings). No se cuelga por sí solo.
- **BUG-018 (crítico) — deslogueo offline.** `AuthContext.loadProfile` consultaba la tabla `users` por red; offline → `user=null` → `AppNavigator` manda a **Login** → bloqueo total sin internet. Anulaba el caché de ruta aguas arriba. **Fix aplicado** (ver [[bugs/Registro de Bugs|BUG-018]]): `userCache.ts` (perfil en `meta`), `loadProfile` red-primero + fallback + timeout 8s, `signOut` limpia caché. 6 tests (38/38 verdes).
- **Instrumentación agregada**: `auth_profile` y `route_load` en `device_logs` para que el próximo retest sea inequívoco.
- **Método de iteración (decisión del usuario)**: rebuild del APK preview cada vuelta (no dev client). Build nuevo: `0648de3e`.
- **Ojo en el retest**: el caché sólo se ejercita si el cold-start es **verdaderamente offline** (en la 1ª prueba había señal al cargar → tomó el camino online, sin banner = correcto).

## ✅ Smoke Test — VALIDADO (build `0648de3e`, sesión offline `29124ad6`)

→ reconectar. Validado por queries a Supabase:

- **`session_end` SET al primer intento** (01:55:24 → 01:57:40) ✔

- **1 visita** (FTD CARICUAO, `completed`) + **1 foto subida a Storage** al reconectar ✔
- **BUG-018 confirmado muerto**: `device_logs` muestra `auth_profile · cache` — offline la app
  restauró el perfil desde caché en vez de ir a Login ✔

**Hallazgo de UX**: "Empezar Ruta" tardó ~15s (fix GPS de alta precisión sin timeout). **Arreglado**:
`utils/withTimeout.ts` (4 tests) — arranque en ~3s con precisión Balanced + timeout; si el fix no
llega, la sesión arranca igual y el tracker completa coords. APK final: `e33a55f5`. 42/42 tests verdes.
